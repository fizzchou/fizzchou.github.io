设计模式入门 Strategy Pattern

使用模式最好的方式是：把模式装进脑子，然后在你的设计和已有的应用中，寻找何处可以使用它们

Duck Class

  • 软件开发不变的真理:CHANGE
  • 设计原则1: 把变化的部分取出来，独立出来，封装起来，从而不影响其他部分

  • 设计原则2: 针对接口编程，而不是针对实现编程(真正的意思是针对超类型编程)

    Animal animal = new dog(); // new cat();
    animal.makeSound //多态

    接口有多个含义，可以不涉及java interface ，关键在于利用多态，真正的意思是：“针对超类型编程”->运行时确定具体行为，而不会绑死在超类的行为上 [P12]

  • 设计原则3: 多组合少继承
    HAS-A can be better than IS-A
  1. 第一个设计模式 Strategy Pattern 策略模式
    定义了算法族，分别封装起来，让它们之间可以互相替换，此模式让算法的变化独立于使用算法的客户
    defines a family of algorithms,encapsulates each one and makes them interchangeable.Strategy lets the algorithm vary independently from clients that use it
  • 使用设计模式交流，多多易善，高效【程序员有自己的共享语言】
  • 编程是一门艺术，设计模式是前人辛苦整理得出的【是oo经验】 具有弹性，易维护，可以应付变化的oo系统才是好系统
  • 了解OO的基础不足以设计出良好的OO系统
  • 好的OO设计是：可复用，易扩展，易维护
  • 设计模式不是被发明的，而是被发现的
  • 我们常常把需要变化的独立出来分装起来

DDScrollProgressView

A lightweight and interactable progress components of iOS

Demo Project

Find Demo/DDScrollProgressView.xcodeproj

Installation

CocoaPods

  1. Add pod 'DDScrollProgressView' to your Podfile.
  2. Run pod install or pod update.
  3. Import correspond header file

Manually

  1. Download all the files in the DDScrollProgressView directory.
  2. Add the source files to you Xocde project
  3. Import correspond header file

Usage

Display default progress

Display with nubmer indicate

Customize Feature

  • Summary look below, detail find to Demo/DDScrollProgressView.xcodeproj

Requirements

This library requires iOS 8.0+ and Xcode 8.0+.
